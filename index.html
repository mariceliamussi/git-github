Introdução ao Git

Git é um sistema de controle de versão distribuído criado por Linus Torvalds em 2005. É uma ferramenta essencial para desenvolvimento de software, permitindo que você:

  • Rastreie mudanças no código
  • Reverta para versões anteriores
  • Trabalhe em equipe eficientemente
  • Mantenha diferentes versões do projeto

Instalação

Windows

Download e instalação no Windows 
1. Acesse https://git-scm.com/download/win
2. Baixe o instalador
3. Execute o arquivo .exe
4. Siga as instruções do instalador

macOS

Instalação via Homebrew 
brew install git

Linux

Instalação via apt (Ubuntu/Debian) 
sudo apt-get update
sudo apt-get install git

Configuração Inicial

Após instalar o Git, você precisa configurar suas credenciais:

Configurações globais 
git config --global user.name "Seu Nome"
git config --global user.email "seu@email.com"
git config --global core.editor "code --wait"  # Para VS Code
git config --global init.defaultBranch main

Estas configurações são armazenadas no arquivo ~/.gitconfig e são aplicadas a todos os repositórios no seu computador.

Repositórios

Um repositório é um diretório onde o Git armazena todos os arquivos do projeto, além de todo o histórico de alterações feitas. Um repositório pode ser local (no seu computador) ou remoto (no GitHub, GitLab, etc.), facilitando o trabalho colaborativo.

Exemplo: para criar um repositório local e conectá-lo a um repositório remoto:

                        
mkdir meu-projeto
cd meu-projeto
git init
git remote add origin https://github.com/usuario/meu-projeto.git

Commits

Commits são "fotografias" do estado atual do seu projeto. Cada commit possui uma mensagem que descreve as mudanças feitas, facilitando o rastreamento do histórico e das versões do projeto.

Exemplo: ao modificar um arquivo, você pode salvá-lo no histórico com:

code>git add arquivo-modificado.txt
git commit -m "Corrigindo o bug na função X"

A mensagem do commit deve ser clara para facilitar a colaboração e o entendimento do histórico.

Branches

Branches são "linhas do tempo" paralelas do seu projeto, permitindo que você trabalhe em novas funcionalidades sem interferir no código principal. Assim, cada branch pode ser desenvolvida e testada de forma independente.

Exemplo: para criar e alternar para uma nova branch:

                        
git branch nova-funcionalidade
git checkout nova-funcionalidade

Ao finalizar o trabalho em uma branch, você pode integrá-la ao código principal (usualmente a branch "main") por meio de um merge.

Merge

Merge é o processo de integrar as mudanças feitas em uma branch para outra, frequentemente combinando a branch de desenvolvimento com a branch principal.

Exemplo: para mesclar a branch "nova-funcionalidade" à branch "main":

                        
git checkout maingit merge nova-funcionalidade

Workflow

Workflow define a metodologia de trabalho com Git, geralmente envolvendo pull requests e revisões de código. O GitHub flow, por exemplo, sugere criar uma branch para cada nova funcionalidade e fazer um pull request para revisão.

Exemplo: no GitHub, você pode criar um pull request para solicitar a integração de uma branch de desenvolvimento à branch principal.

# Crie uma nova branch
git checkout -b nova-funcionalidade

# Após as mudanças, envie para o repositório remoto
git push origin nova-funcionalidade

# Abra um Pull Request no GitHub para revisão

Conflitos

Conflitos ocorrem quando duas branches possuem alterações incompatíveis no mesmo arquivo. O Git marca as diferenças para que você resolva manualmente e determine quais mudanças serão mantidas.

Exemplo: ao ocorrer um conflito, edite o arquivo conforme necessário e faça um novo commit:

# Após resolver o conflito, adicione e faça o commit
git add arquivo-resolvido.txt
git commit -m "Resolvendo conflito no arquivo"

Rebase

O comando rebase reorganiza o histórico de commits, útil para manter uma linha do tempo linear. Ao fazer um rebase, você pode "reaplicar" os commits da sua branch em uma nova base, como a branch principal.

Exemplo: para fazer o rebase da sua branch na branch "main":

git checkout minha-branch
git rebase main

Tags

Tags são usadas para marcar versões importantes no histórico, como lançamentos ou milestones do projeto.

Exemplo: para criar uma tag de versão 1.0 e enviar para o repositório remoto:

git tag v1.0
git push origin v1.0

Introdução ao GitHub

GitHub é uma plataforma de hospedagem de código baseada em Git, que permite colaboração entre desenvolvedores, além de oferecer ferramentas para o controle de versão e desenvolvimento em equipe.

Exemplo: Para enviar um projeto local ao GitHub, crie um repositório no site e conecte-o ao repositório local:

git remote add origin https://github.com/usuario/projeto.git
git push -u origin main

Pull Requests

Pull Requests (PRs) permitem que você proponha alterações ao repositório. Eles são revisados antes de serem integrados, garantindo a qualidade do código.

Exemplo: Após fazer push de uma nova branch para o GitHub, acesse o repositório online e clique em "New Pull Request" para iniciar uma revisão.

# Crie uma nova branch, faça as alterações e envie para o GitHub
git checkout -b minha-feature
git push origin minha-feature

# Em seguida, crie um Pull Request no GitHub

Issues

Issues são usadas para rastrear problemas, bugs e solicitações de funcionalidades no projeto. Cada issue pode ser comentada e referenciada em commits e PRs, facilitando o gerenciamento do projeto.

Exemplo: Para criar uma nova issue, vá até a aba "Issues" do repositório no GitHub e clique em "New Issue".

Actions

GitHub Actions permitem a automação de fluxos de trabalho, como testes automatizados e deploy. Você define workflows no repositório para executar tarefas automaticamente a cada push ou PR.

Exemplo: Para configurar um workflow, adicione um arquivo YAML na pasta .github/workflows:


name: CI
on: [push, pull_request]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Configurar Node.js
        uses: actions/setup-node@v2
        with:
        node-version: '14'
        run: npm install
      - run: npm test

Esse exemplo executa testes automáticos a cada novo push ou pull request no repositório.
